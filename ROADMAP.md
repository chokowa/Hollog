# Hollog 開発ロードマップ

現在の「ローカル完結型Webアプリ」としての強み（爆速な開発スピードと修正力）を最大限に活かしつつ、最終的に動画などの制限をなくした「完全なネイティブアプリ」へと進化させるための段階的なロードマップです。

> 運用メモ: PC Web版 / PWA / Vercel運用は今後使う可能性が低いため凍結扱い。以後はAndroidネイティブ / Capacitorアプリを優先し、Web版固有の対処や同期は明示依頼がない限り行わない。

## Phase 1: Webアプリ版の完成・極限のブラッシュアップ（実用版完了）
**目的：開発スピードの最速化とUI/UXの完成度向上**

Phase 1 は、ネイティブ化前の機能検証としては完了扱いにします。細部のUI/UX調整は、Android実機での操作感を見ながらPhase 2以降で継続します。

- [x] 複数画像の投稿・保存機能（ IndexedDB への配列保存 ）
- [x] 高品質な画像表示レイアウト（Xのようなグリッドクロップ、全画面画像ビューア）
- [x] 設定（Settings）画面の実装
  - [x] タグのサジェスト候補の追加・編集・削除機能
  - [x] タグ候補一覧（▼から開いた時のリスト）の並び替え機能
  - [x] テーマ切り替え、投稿済みカード表示設定
- [x] 検索・タグ絞り込みの基本機能
  - 本文・URL・OGP情報・タグを対象にした検索
  - タブ内タグ一覧とタグクリックによる絞り込み
- [x] カレンダー表示
  - 日付ごとの投稿確認
  - ポスト / クリップの絞り込み
- [x] タグ管理・一括タグ付け
  - 専用のタグ管理画面
  - 複数投稿へのタグ追加・置換
  - タグ更新時はタイムライン順を維持する
- [ ] UI/UX の完全な洗練（ネイティブ化後に実機操作を見ながら継続）
- [x] タイムライン画像のサムネイルキャッシュ化
  - [x] 投稿時に軽量サムネイルを生成・保存する
  - [x] タイムラインはサムネイル、画像ビューア/詳細は元画像を使う
  - ネイティブ化後は、端末内に元ファイルがある画像/動画は元ファイル参照、Web/一時共有など元が端末内にないものはアプリ管理、サムネイルだけ軽量キャッシュにする
- [ ] 未実装機能の補完
  - プロフィール機能は方針未定。自己紹介用途は不要な可能性が高く、削除も含めて検討する
  - 検索・タグ管理は基本実装済み。必要に応じて履歴、保存条件、タグ整理の操作感を追加調整する
- [ ] PWA（ホーム画面追加）による疑似アプリとしての運用テスト
  - [x] Web App Manifest の基本設定
  - [x] ホーム画面追加用アイコンの追加
  - [ ] ホーム画面追加の実機確認
  - [ ] オフライン動作やキャッシュ戦略の要否を判断

*※このフェーズでは、動画対応による容量問題やビルドエラーのトラブルを避け、Web技術のメリット（ホットリロード）をフル活用して機能を作り込みます。*

## Phase 3: ネイティブアプリ化の完了と制限の解放（最終）

## Next: 直近で追加したい機能メモ
**目的：日常利用の気持ちよさを上げつつ、Hollog を「自分用メディア保管庫 / 下書き置き場」として強くする**

### 方針メモ
- Hollog の主役はタイムラインとし、X に慣れた人が「こうだったらよかったのに」と感じる体験をローカル前提で補う
- タイムライン / カレンダー / メディアは別機能ではなく、同じ投稿を別の見方で見せる表示モードとして扱う
- 設定は極力増やさず、使い勝手で意見が分かれるところだけを少数の切り替えに絞る
- ロケーションのような私的でオンライン保存が怖い情報も、ローカル保存前提で安心して残せるようにする
- 投稿画面は自動保存前提に寄せすぎず、不要な投稿を気軽に破棄できるよう保存 / 投稿の明示操作を残す方向で考える
- 自動保存を導入する場合はデフォルトではなく、クイック保存モードのような任意機能として扱うことを検討する
- 画像投稿は「保存した日 / 投稿した日」をタイムライン上の基準とし、「撮影日 / 元データ日」は別メタデータとして保持する方向で考える
- 旅行写真や収集画像をシール帳のように近くへ置く使い方を尊重し、写真アプリのように過去日へ自動配置しすぎない

- [ ] タグヒートマップ
  - 一目でどのタグが多いか分かる可視化を作る
  - ヒートマップから見たいタグだけを残してフィルタできるようにする
- [x] HOME 上部タグの長押しからメディア表示へ遷移
  - HOME画面上部のタグを長押しすると、メディアタブでそのタグだけに絞り込んだ状態へ入る
  - 画像系タグを多用している場合に、アルバムを見る感覚でピンタレスト表示や連続表示を楽しめるようにする
- [ ] HOME 画面の表示密度切り替え
  - 現在の1列表示に加えて、2列表示と3列表示を切り替えられるようにする
- [ ] HOME カード上のコメント表示文字数の調整
  - HOME画面のカードで、本文をどこまで表示するかの文字数 / 行数制限を見直せるようにする
  - 一覧での見やすさと、情報量のバランスを調整できるようにする
- [ ] 下書きの自動保存
  - 入力途中の内容を保持し、閉じても途中状態から再開できるようにする
- [ ] アーカイブ / ゴミ箱
  - 削除以外に、いったん隠す / あとで見返すための退避先を用意する
- [ ] 並び替え
  - 更新順以外に、作成順や古い順などを選べるようにする
- [ ] ピン留め機能
  - いつも見たい投稿を先頭に置けるようにする
  - ピン留めされた投稿は通常の更新順とは別に扱う
- [ ] 日付ジャンプ
  - 特定の日付の投稿へすばやく移動できる導線を用意する
- [ ] X 導線のデフォルト非表示化
  - X への投稿機能を常時見せる前提から外す
  - 外部SNS全般に使いやすい位置付けに寄せる
  - 設定から X へのポスト導線を表に出せるようにし、普段は隠しておく
- [x] JSON でのエクスポート / インポート
  - ポストとクリップを JSON 形式で書き出し / 復元できるようにする
  - v1 は投稿本文、URL、タグ、OGP、表示設定、メディア参照メタデータを対象にする
  - v1 のインポートでは重複追加を避け、既存本文を保持し、タグだけ結合する
  - 本文 / URL / OGP に差分がある重複投稿は、既存保持 / インポート版 / 個別選択を確認してから適用する
- [ ] メディアのエクスポート / インポート
  - ローカル参照している画像を一括コピーし、リンクを壊さない形でエクスポート / インポートできるようにする
  - ローカル参照の投稿が多い人向けに、保存先の移動や取り扱いについて注意が必要なことを明示する
  - オンボーディングで使う保存方針の説明候補: 「撮影画像やコピー＆ペーストした画像は、Hollog内に保存されます。画像メニューから保存すると、外部フォルダへコピーされ、写真アプリなどからも見られるようになります。」
- [ ] 復元を含めた入出力の安全性
  - 機種変更時の利用を想定し、インポート / エクスポート時の確認、重複時の扱い、失敗時の安全性を整理する
